字数统计: 485字 | 阅读时长 ≈ 2分钟


看了很多网页现在开始用HEXO,简洁明了，打算开使搭建HEXO
，开始爬坑之路!

尚未解决的问题：

  1. 部署到阿里云服务器，服务器是去年买的，前面一段时间是用PHP编写，感觉他的可用性没有HEXO高。
  2. 由于时间的关系，暂时托管再Github
  3. 图片没办法显示
  4. 标签没办法连接到文章
    走过的坑：

  5. 更新主题没有反应
    解决办法：
    先hexo clean 在hexo g -d试试
    清空public 效果就是hexo clean 相当于把public静态文件清空在重新生成部署。
    hexo generate hexo deploy 简写 hexo g -d

  6. 设置SHH密钥时出现错误显示：connect to host github.com port 22: Connection timed out。
    解决办法：
    在.ssh目录下创建一个config文件，
    输入如下内容： Host github.com User xxx@qq.com （你注册github时的邮箱，这里使用注册的用户名也行） Hostname ssh.github.com PreferredAuthentications publickey IdentityFile ~/.ssh/id_rsa Port 443 值得注意是文件名为config，没有后缀名。 然后测试是否连得上 :
    即：ssh -T git@github.com 会
    出现以下情况：
    出现Hi xxx!……表示连接成功。

  7. github 网页显示不带CSS样式
    安装hexo-deployer-git自动部署发布工具 ,
    ==npm instal lhexo-deployer-git –save== ，这个工具提供一些简单的指令帮助你发布博文到博客，比如每次修改了或者新增了博客，只用==hexo generate== 生成静态文件，然后==hexo deploy==就可以发布到github page。
    安装成功以后，依次运行==hexo clean hexo generate hexo deploy。==
